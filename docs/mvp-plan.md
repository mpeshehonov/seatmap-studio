# Seatmap Studio MVP Plan

## Цель

Собрать бесплатный прототип замены seats.io на Next.js + Supabase + Vercel: админка для создания площадок и залов, простой редактор схемы, публичный embed-виджет выбора мест.

## Архитектура

Один Next.js App Router проект содержит админку, публичный widget и route handlers. Supabase закрывает Auth, Postgres и RLS. Vercel деплоит фронтенд и API routes без отдельного сервера.

## Этап 1: Каркас и данные

- Создать Next.js проект `seatmap-studio`.
- Подключить `@supabase/ssr` и `@supabase/supabase-js`.
- Добавить `.env.example` с `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`.
- Описать SQL-схему `profiles`, `venues`, `halls`, `seat_maps`, `events`, `seat_statuses`.
- Включить RLS на всех публичных таблицах.
- Добавить демо JSON схемы зала, чтобы проект работал без реальных Supabase ключей.

## Этап 2: Админка

- `/login`: magic link вход через Supabase Auth.
- `/dashboard`: список площадок и залов текущего аккаунта.
- `/venues`: создание и редактирование площадок.
- `/halls/[hallId]/editor`: редактор схемы зала.
- В первой версии редактор поддерживает прямые ряды, места и простые фигуры типа сцены.
- Сохранение идет в `seat_maps.map_json`.

## Этап 3: Embed-виджет

- `/embed/[hallId]`: публичная страница для iframe.
- `/api/embed/halls/[hallId]`: публичная JSON-выдача схемы и статусов.
- Виджет рисует места, показывает `available`, `held`, `sold`, позволяет выбрать доступные места.
- Для MVP выбранные места отображаются в UI; полноценный hold/reservation flow добавляется после проверки спроса.

## Этап 4: Интеграция с реальной базой

- Создать проект Supabase.
- Выполнить `supabase/schema.sql` в SQL Editor.
- Добавить env-переменные в `.env.local` и Vercel.
- Заменить демо-данные на запросы к Supabase в dashboard/editor/embed API.
- Проверить, что аккаунт видит только свои площадки и залы, а публичный embed читает только опубликованные залы.

## Этап 5: Полировка прототипа

- Добавить операции редактора: создать ряд, удалить ряд, изменить цену, изменить категорию.
- Добавить импорт/экспорт JSON.
- Добавить простую мобильную адаптацию widget.
- Проверить схему на реальном зале 100-300 мест.

## Что намеренно не входит в MVP

- Криволинейные ряды.
- Сложные трансформации секторов.
- Стадионы 10k+ мест.
- Платежи и полноценное бронирование.
- Marketplace виджетов и white-label настройки.
